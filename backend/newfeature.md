The new feature to be implemented is an AI-powered Repository Setup Documentation Generator that uses the Gemini API/Grock as a central component of the OpenSource Compass platform. The purpose of this feature is to help beginners and first-time contributors easily understand how to set up and run any recommended GitHub repository locally, since many open-source projects have complex, incomplete, or overly technical README files. When a user selects a repository (for example nodejs/node), they should be able to request an instant setup guide through a new Core Service endpoint such as GET /repo/setup-guide?repo=owner/repo&user_id=<id> with JWT authentication. The Core Service will call the GitHub microservice to fetch repository metadata and the full README content, bundle this information into a single well-structured Gemini/grock prompt, and send it to the AI Service running Gemini/grock so that only one AI call is made per request (to avoid free-tier rate limit issues). Gemini/grock must return a structured beginner-friendly setup guide that includes prerequisites, installation steps, run commands, environment variables if required, common errors with fixes, and contribution notes such as how to start with good first issues. The output should be personalized based on the user’s experience level stored in the database (beginner users receive more explanation, intermediate users receive concise steps, advanced users receive mostly commands). The final response should be returned as clean JSON so the frontend can display it in a readable format. The complete backend flow should work as: client calls Core Service → Core calls GitHub Service for README and repo details → Core builds one combined Gemini/grock request → AI Service calls Gemini/grock once → Gemini/grock generates simplified setup documentation → Core returns the guide to the user. This feature adds strong hackathon value because Gemini/grock becomes essential in transforming raw repository documentation into clear onboarding instructions, enabling users to move from repository discovery to real contribution much faster.
