The Problem: The Implementation Gap

Vague Issues

Missed Opportunities

"Good First Issues" are snapped up in minutes. You can't manually refresh GitHub 24/7.

1

AI-Powered Context

2

Personalized Matching

Matches filtered strictly by your tech stack (e.g., "Go" + "Beginner"). No noise.

3

Real-Time User Flows

WebSocket alerts the moment a relevant issue drops. Never miss out.

Deep Dive: The AI Engine

The AI Service acts as a translation layer, converting raw GitHub issue data into structured developer guides.

Provider Agnostic

Seamlessly switches between Google Gemini and Groq (Llama 3) using environment variables.

Prompt Engineering

We inject a "Senior Developer Persona" into the prompt to force the LLM to output structured, actionable steps.

Structured Output

Returns strict JSON with difficulty, steps, and required_skills for frontend rendering.

The Watchlist Engine

How we ensure users never miss an opportunity.

The Poller (Core Service)

A dedicated Go routine that monitors all watched repositories concurrently. It tracks the `latest_issue_number` to detect deltas.

Notification Service

Manages active WebSocket connections. When the Poller triggers, this service instantaneously pushes data to all connected clients.

Future Roadmap

Phase 1: RepoChat (RAG)

"Chat with the codebase." We will use Vector Embeddings (Pinecone) to let users ask "Where is the auth logic defined in this repo?" and get code pointers.

Phase 2: Gamification

Introduction of XP bars, "Bug Hunter" badges, and Leaderboards. This drives user retention and encourages "contribution streaks".

Phase 3: Auto-Setup

AI generates a custom docker-compose.yml or setup.sh for *any* repo based on its README, solving the "it works on my machine" problem.
